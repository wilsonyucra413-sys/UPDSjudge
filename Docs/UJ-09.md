# UJ-09 - Carga Automática de Casos de Prueba

## Descripción

Permite cargar automáticamente los casos de prueba mediante un archivo ZIP organizado por carpetas de incisos.

---

# Endpoint

Forma parte del endpoint:

**POST** `/api/concursos/crear`

---

# Validaciones del Archivo ZIP

## 1. Archivo obligatorio

Debe enviarse un archivo ZIP.

---

## 2. Extensión

El archivo debe tener extensión:

```
.zip
```

---

## 3. Tamaño máximo

El archivo ZIP comprimido no puede superar:

**100 MB**

---

## 4. Tamaño descomprimido

El contenido total descomprimido no puede superar:

**600 MB**

---

## 5. Tamaño máximo por archivo

Cada archivo individual no puede superar:

**5 MB**

---

## Validaciones de la Estructura

Cada problema debe tener una carpeta con su inciso.

Ejemplo:

```text
A/
B/
C/
```

---

No pueden existir carpetas que no hayan sido declaradas como problemas.

Ejemplo:

Problemas registrados:

```
A
B
```

ZIP:

```
A
B
C
```

Resultado:

Error.

---

No pueden faltar carpetas.

Ejemplo:

Problemas registrados:

```
A
B
C
```

ZIP:

```
A
B
```

Resultado:

Error.

---

Cada carpeta debe contener al menos un archivo:

```
.in
```

---

# Validaciones de Casos de Prueba

Para cada archivo:

```
archivo.in
```

debe existir exactamente su correspondiente:

```
archivo.out
```

Ejemplo:

```
1.in
1.out
```

---

No se aceptan archivos `.in` sin su correspondiente `.out`.

---

No se aceptan archivos `.out` sin su correspondiente `.in`.

---

Los archivos `.in` no pueden estar vacíos.

---

Los archivos `.out` no pueden estar vacíos.

---

Cada inciso debe contener al menos un caso de prueba válido.

---

# Procesamiento

Si todas las validaciones son correctas:

1. Se descomprime el archivo.
2. Se valida la estructura.
3. Se validan todos los casos de prueba.
4. Se lee el contenido de cada archivo.
5. Los contenidos son almacenados en la base de datos asociados a su problema correspondiente.

---

# Respuestas Posibles

| HTTP | Motivo |
|------|--------|
|200|Casos cargados correctamente.|
|400|ZIP inválido.|
|400|Estructura incorrecta.|
|400|Faltan carpetas.|
|400|Sobran carpetas.|
|400|Faltan archivos `.out`.|
|400|Archivo vacío.|
|400|Se excedió el tamaño permitido.|

---
## Formato de codigo 
| Tipo                     | Código          |
| ------------------------ | --------------- |
| Entrenamiento Div4       | `UPDS-DIV4-001` |
| Entrenamiento Div3       | `UPDS-DIV3-001` |
| Entrenamiento Div2       | `UPDS-DIV2-001` |
| Entrenamiento Div1       | `UPDS-DIV1-001` |
| Clasificatoria           | `UPDS-PREV-001` |
| Competencia oficial      | `UPDS-001`      |
| Competencia ICPC interna | `UPDS-ICPC-001` |
| Maratón especial         | `UPDS-SP-001`   |

# Reglas de Negocio

- Solo se aceptan archivos ZIP.
- La estructura debe coincidir exactamente con los problemas registrados.
- Cada problema debe poseer al menos un caso de prueba.
- Todos los casos deben estar formados por un archivo `.in` y un archivo `.out`.
- Los archivos vacíos no son válidos.
- El proceso completo forma parte de la creación del concurso; si falla cualquier validación, no se almacena ningún dato.
