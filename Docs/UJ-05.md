# UJ-05 - Registro de Usuario

## Descripción
Permite registrar un nuevo usuario en la plataforma mediante nombre, correo electrónico y contraseña.

---

# Endpoint

**Método:** `POST`

**Ruta:** `/api/auth/register`

---

# Datos de Entrada

| Campo | Tipo | Obligatorio | Descripción |
|--------|------|-------------|-------------|
| nombre | string | Sí | Nombre del usuario. |
| correo | string | Sí | Correo electrónico del usuario. |
| contrasena | string | Sí | Contraseña del usuario. |

---

# Validaciones

## 1. Campos obligatorios

Se valida que los siguientes campos estén presentes y no estén vacíos:

- nombre
- correo
- contrasena

### Si la validación falla

**HTTP:** `400 Bad Request`

```json
{
    "mensaje": "Nombre, correo y contraseña son obligatorios."
}
```

---

## 2. Correo único

Antes de crear el usuario se verifica que el correo electrónico no exista previamente en el sistema.

### Si el correo ya existe

**HTTP:** `400 Bad Request`

```json
{
    "mensaje": "El correo ya está registrado."
}
```

---

# Procesamiento

Si todas las validaciones son correctas se realizan las siguientes acciones:

1. Se eliminan espacios al inicio y final del nombre.
2. Se eliminan espacios al inicio y final del correo.
3. La contraseña se almacena utilizando un algoritmo de hash seguro (BCrypt).
4. Se registra la fecha y hora del registro en UTC.
5. El usuario es creado con estado **Activo**.
6. Se asigna automáticamente el rol **Usuario**.
7. Se guarda la información en la base de datos.

---

# Respuesta Exitosa

**HTTP:** `201 Created`

```json
{
    "mensaje": "Usuario registrado exitosamente.",
    "correo": "usuario@correo.com"
}
```

---

# Respuestas Posibles

| Código HTTP | Motivo |
|-------------|--------|
| 201 Created | Usuario registrado correctamente. |
| 400 Bad Request | Existen campos obligatorios vacíos. |
| 400 Bad Request | El correo ya se encuentra registrado. |

---

# Reglas de Negocio

- Todos los usuarios nuevos son registrados con estado **Activo**.
- Todo usuario nuevo recibe automáticamente el rol **Usuario**.
- No pueden existir dos usuarios con el mismo correo electrónico.
- La contraseña nunca se almacena en texto plano.
- La fecha de registro se almacena en formato UTC.
