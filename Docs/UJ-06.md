# UJ-06 - Inicio de Sesión

## Descripción

Permite que un usuario registrado inicie sesión en la plataforma y obtenga un token JWT que contiene su información de identificación y sus roles para acceder a los recursos protegidos.

---

# Endpoint

**Método:** `POST`

**Ruta:** `/api/auth/login`

---

# Datos de Entrada

| Campo | Tipo | Obligatorio | Descripción |
|--------|------|-------------|-------------|
| correo | string | Sí | Correo electrónico registrado del usuario. |
| contrasena | string | Sí | Contraseña del usuario. |

---

# Validaciones

## 1. Campos obligatorios

Se valida que los siguientes campos estén presentes y no estén vacíos:

- correo
- contrasena

### Si la validación falla

**HTTP:** `400 Bad Request`

```json
{
    "mensaje": "Correo y contraseña son obligatorios."
}
```

---

## 2. Existencia del usuario

Se verifica que exista un usuario registrado con el correo electrónico proporcionado.

### Si la validación falla

**HTTP:** `401 Unauthorized`

```json
{
    "mensaje": "Credenciales inválidas."
}
```

> **Nota:** No se informa si el correo existe o no, con el fin de evitar la enumeración de usuarios.

---

## 3. Verificación de contraseña

Se valida que la contraseña ingresada coincida con la contraseña almacenada de forma segura en la base de datos.

### Si la validación falla

**HTTP:** `401 Unauthorized`

```json
{
    "mensaje": "Credenciales inválidas."
}
```

> **Nota:** Se utiliza el mismo mensaje de error para impedir revelar cuál de las credenciales es incorrecta.

---

# Procesamiento

Si todas las validaciones son correctas se realizan las siguientes acciones:

1. Se recupera la información del usuario.
2. Se obtienen todos los roles asignados al usuario.
3. Se genera un token JWT firmado.
4. El token incluye la información necesaria para la autenticación y autorización.
5. Se establece una fecha de expiración para el token de **8 horas** desde su generación.
6. Se devuelve el token al cliente para su uso en las solicitudes autenticadas.

---

# Información incluida en el JWT

El token generado contiene los siguientes datos:

| Claim | Descripción |
|--------|-------------|
| idUsuario | Identificador único del usuario. |
| nombre | Nombre del usuario. |
| correo | Correo electrónico del usuario. |
| Name | Nombre estándar utilizado por el framework de autenticación. |
| Email | Correo estándar utilizado por el framework de autenticación. |
| Role | Uno o más roles asignados al usuario. |

---

# Respuesta Exitosa

**HTTP:** `200 OK`

```json
{
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "expiraEn": "2026-07-20T23:15:42Z"
}
```

---

# Respuestas Posibles

| Código HTTP | Motivo |
|-------------|--------|
| 200 OK | Inicio de sesión exitoso. |
| 400 Bad Request | Existen campos obligatorios vacíos. |
| 401 Unauthorized | Credenciales inválidas. |

---

# Reglas de Negocio

- Solo los usuarios registrados pueden iniciar sesión.
- El correo y la contraseña son obligatorios.
- No se revela si el correo existe o no en el sistema.
- La contraseña se verifica utilizando su hash almacenado.
- El token JWT tiene una vigencia de **8 horas**.
- El token incluye todos los roles asignados al usuario.
- El token debe utilizarse para acceder a los endpoints protegidos mediante autenticación.
- El token está firmado digitalmente para garantizar su integridad y autenticidad.
