# UJ-08 - Crear Concurso

## Descripción

Permite a un Administrador de Concursos crear un nuevo concurso indicando su información general, configuración y lista de problemas.

---

# Endpoint

**Método:** `POST`

**Ruta:** `/api/concursos/crear`

**Autorización requerida**

- JWT válido.
- Rol **AdministradorConcursos**.

---

# Datos de Entrada

| Campo | Tipo | Obligatorio |
|--------|------|-------------|
| codigo | string | Sí |
| nombre | string | Sí |
| descripcion | string | Sí |
| fechaInicio | datetime | Sí |
| duracionMinutos | int | Sí |
| minutosCongelamiento | int | Sí |
| contrasena | string | No |
| urlSetProblemas | string | Sí |
| listaProblemas | List | Sí |
| archivoZip | File (.zip) | Sí |

---

# Validaciones

## 1. Autenticación

El usuario debe enviar un JWT válido.

### Si falla

**HTTP:** `401 Unauthorized`

```json
{
    "mensaje":"Token inválido"
}
```

---

## 2. Autorización

Solo usuarios con el rol:

- AdministradorConcursos

pueden crear concursos.

---

## 3. Código del concurso

Se valida que:

- sea obligatorio.
- cumpla el formato:

```
UPDS-001
```

o

```
UPDS-DIV4-001
UPDS-DIV3-001
UPDS-DIV2-001
UPDS-DIV1-001
UPDS-PREV-001
UPDS-ICPC-001
UPDS-SP-001
```

Reglas:

- debe comenzar con `UPDS`
- la categoría es opcional
- la categoría solo puede ser:
  - DIV4
  - DIV3
  - DIV2
  - DIV1
  - PREV
  - ICPC
  - SP
- debe terminar exactamente con tres dígitos.

---

## 4. Código único

No puede existir otro concurso activo con el mismo código.

---

## 5. Nombre

Debe estar presente.

---

## 6. Descripción

Debe estar presente.

---

## 7. Fecha de inicio

Se valida que:

- exista.
- sea posterior a la fecha actual.
- no supere los 12 meses de anticipación.

---

## 8. Duración

Debe:

- ser mayor a 0 minutos.
- no superar 7 días.

---

## 9. Minutos de congelamiento

Debe:

- ser mayor o igual a 0.
- ser menor que la duración del concurso.

---

## 10. Problemas

Debe existir al menos un problema.

No puede haber más de **12 problemas**.

Cada problema debe cumplir:

- inciso válido (A-Z)
- título obligatorio
- tiempo mayor a 0
- memoria mayor a 0

No pueden existir incisos repetidos.

---

# Procesamiento

Si todas las validaciones son correctas:

1. Se crea el concurso.
2. Se registra el usuario creador.
3. Se crean todos los problemas.
4. Se crean posteriormente los casos de prueba.
5. Toda la operación se realiza dentro de una transacción.

Si ocurre un error durante el proceso se revierte completamente la operación.

---

# Respuesta Exitosa

**HTTP:** `200 OK`

```json
{
    "codigo":"upds-div4-001",
    "mensaje":"Concurso, problemas y casos de prueba creados exitosamente."
}
```

---

# Respuestas Posibles

| HTTP | Motivo |
|------|--------|
|200|Concurso creado correctamente.|
|400|Alguna validación de negocio falló.|
|401|Token inválido.|
|403|Usuario sin permisos.|

---

# Reglas de Negocio

- Solo un Administrador de Concursos puede crear concursos.
- Los códigos deben ser únicos entre concursos activos.
- Un concurso puede tener máximo 12 problemas.
- Cada problema debe tener un inciso único.
- El concurso se crea únicamente si todas las validaciones son exitosas.
- La creación es atómica; si falla cualquier paso, no se almacena información parcial.
